# ═══════════ ТЗ v1.1: MMO Космічна Стратегія для Habitat OS  
**Дата:** 27 травня 2026  
**Платформа:** Habitat OS (web-based, mobile-first, 320px viewport)  
**Жанр:** MMO космічна стратегія у стилі Travian / Ogame  
**Цільова аудиторія:** гравці 16–35 років, досвідчені у стратегіях, активні у мобільних іграх  
**Мета:** створити глибоку, балансну, соціально-орієнтовану космічну стратегію з поступовим прогресом, альянсами, дослідженнями та боєм у реальному часі.  

---

## 1. Загальний опис

Гравець керує космічною колонією на планеті, розвиває інфраструктуру, добуває ресурси, будує флот, досліджує технології, вступає в альянси та бере участь у міжзоряних конфліктах. Гра працює у фоні (real-time), але взаємодія — через інтерфейс Habitat OS. Усі екрани оптимізовані під мобільні пристрої (320px).  

---

## 2. Інтеграція з Habitat OS

### 2.1 Навігація
До існуючого нижнього меню Habitat OS додається 5 нових кнопок для MMO:
```
[🏠] [🌌] [🚀] [🤝] [🔬]
 Планета Галактика Флот Альянс Дослідження
```
Нові екрани додаються як `<div id="screen-planet" class="screen">...</div>` в index.html.

### 2.2 Економіка
MMO використовує існуючі ресурси Habitat OS:
- **Gold** — основна валюта MMO: будівництво, флот, дослідження
- **Ruby** — преміум-валюта: прискорення, спеціальні товари
- **Sapphire** — рідкісна: VIP, альянси, розширення лімітів
- **Pixel-hours** — паливо для флоту + час активності

Депозит/зняття відбувається прямо з SAVE.resources.

### 2.3 Магазин
Існуючий магазин Habitat OS розширюється новою категорією "Космос" з товарами для MMO.

### 2.4 GitHub синхронізація
Нові data/ файли створюються за тим самим патерном GitHub API:
- data/players.json — реєстрація гравців
- data/planets.json — стан планет
- data/attacks.json — черга атак
- data/alliances.json — альянси
- data/messages.json — повідомлення
- data/reports.json — звіти боїв

### 2.5 Збереження
MMO РОЗШИРЮЄ існуючий save_state (localStorage: habitat_os_save), додаючи поле `SAVE.mmo` зі станом планети, флоту, досліджень (див. розділ 9).

## 7. Бойова система

### 7.1 Механіка бою
- Бої відбуваються за системою "атака vs захист"  
- Кожен корабель має: **атаку**, **захист**, **життя** (життя = захист × 10)  
- Бої розраховуються за формулою:  
  ```
  Урон = Атака - (Захист * 0.5)
  Якщо урон ≤ 0 → мінімальний урон = 1
  ```

### 7.2 Приклад бою

**Гравець А (атакує):**  
- 5 Легких винищувачів (Атака: 4, Захист: 10)  
- Загальна атака: 5 × 4 = 20  

**Гравець Б (захищається):**  
- 3 Легких винищувачів (Захист: 10)  
- 10 Зенітних гармат (Захист: 20, статичний захист)  

> **Ефективний захист Гравця Б:**  
> - Від флоту: 3 × 10 = 30  
> - Від наземних атак: +10 × 20 = 200 (але не впливає на космічні бої)  
> → Загальний захист у бою: 30  

> **Урон від Гравця А:**  
> 20 (атака) - (30 × 0.5) = 20 - 15 = **5 урону**  
> → Розподіляється по кораблях Гравця Б  

> Кожен корабель має 100 життів (захист × 10).  
> 5 урону — недостатньо, щоб знищити жоден корабель.  

**Результат:**  
- Гравець Б втрачає 0 кораблів  
- Гравець А повертається з поразкою  
- Жоден ресурс не здобуто  

> Якби у Гравця А було 20 Легких винищувачів (атака 80):  
> Урон = 80 - 15 = 65 → знищено 1 корабель (100 життів, але урон накопичується)  
> Після 2 таких атак — 2 кораблі знищено

#### `data/combat_rules.js`
```javascript
function calculateDamage(attackerAttack, defenderDefense) {
  let damage = attackerAttack - (defenderDefense * 0.5);
  return damage <= 0 ? 1 : Math.floor(damage);
}

function simulateBattle(attackerFleet, defenderFleet) {
  let defenderHP = defenderFleet.map(ship => ship.defense * 10);
  let totalAttack = attackerFleet.reduce((sum, s) => sum + s.attack, 0);
  let damage = calculateDamage(totalAttack, defenderFleet.reduce((sum, s) => sum + s.defense, 0));
  
  // Розподіл урону
  let shipsDestroyed = 0;
  for (let i = 0; i < defenderHP.length; i++) {
    if (damage >= defenderHP[i]) {
      shipsDestroyed++;
      damage -= defenderHP[i];
    }
  }
  return { attacker: attackerFleet, defender: defenderFleet, lost: shipsDestroyed };
}
```
