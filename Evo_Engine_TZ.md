# ТЕХНІЧНЕ ЗАВДАННЯ: ГЕЙМІФІКОВАНИЙ ІГРОВИЙ РУШІЙ «EVO-ENGINE»

**Проект:** Evo-Engine  
**Репозиторій:** `fil-m/habitat-os-workshop`  
**Стек:** HTML / CSS / Vanilla JS (SPA/PWA на GitHub Pages, без фреймворків)  
**Цільова платформа:** 📱 Mobile-First (мінімум 375px), десктоп адаптація  
**Версія:** 2.0

## 1. Концепція Продукту та УТП

**Evo-Engine** — це гібрид веб-редактора мікро-ігор та мета-гри. Користувачі виступають одночасно гравцями та творцями, розвиваючи свій інструментарій ("еволюціонуючи" рушій) за допомогою зароблених балів.

**Ключові особливості:**
1. **Еволюційний доступ:** Розробник починає з найпростішого 0-1D функціоналу. За зароблені в іграх бали еволюції (EVO) він розблоковує нові виміри (2D, 3D), кольори, фізику та модулі.
2. **Орієнтація на мікро-масштаб:** Рушій свідомо розрахований на малу кількість об'єктів (до 1000 елементів у 3D, до 100 активних сутностей у 2D/1D). Це забезпечує високу продуктивність (через CSS 3D та DOM) і фокусує творців на геймплеї, а не масштабі.
3. **Єдина логічна база:** Всі ігри, незалежно від виміру, працюють на спільній системі змінних та подій.

## 2. Система Відображення (View System)

Рушій підтримує три фундаментальні варіанти візуалізації, які відкриваються по мірі еволюції користувача. Усі вони описують одну й ту саму сцену, але по-різному її рендерять:

### 2.1. Етап 0-1D: Вигляд згори (Top-Down View)
* **Концепт:** Базовий рівень. Камера зафіксована строго зверху. 
* **Геймплей:** Текстові квести, інтерактивні карти, базові лабіринти, головоломки, 2D-аркади без гравітації (як Pacman або ранні Zelda).
* **Рендер:** Сітка (grid), де об'єкти займають конкретні координати [x][y]. Фізика гравітації по осі Z відсутня.

### 2.2. Етап 2D: Вигляд збоку (Side-Scroll View)
* **Концепт:** Класична двовимірна площина з урахуванням осі висоти. Камера зафіксована збоку.
* **Геймплей:** Платформери (типу Mario), side-scroller шутери.
* **Рендер:** Додається постійний вектор гравітації вниз. З'являється фізика стрибка (Jump Physics) та направлена колізія (верх/низ/боки).

### 2.3. Етап 3D: Всі можливі ракурси (Omni-Directional / 3D View)
* **Концепт:** Повноцінний просторовий рендер на базі вокселів.
* **Геймплей:** 3D-платформери, просторові головоломки, симулятори ходьби.
* **Рендер:** Користувач може використовувати різні камери:
  * **Орбітальна камера (Orbital):** Обертання навколо центру сцени (ізометрія, огляд об'єкта зі всіх боків).
  * **Від першої особи (FPS):** Вільний рух камерою всередині світу.
  * **Вільна камера (Free/Fly):** Для зручного редагування воксельного простору зі всіх ракурсів.

## 3. Архітектура: Модульна система (Feature Flags)

Движок є єдиним організмом. Нові функції — це "модулі", які активуються в State-менеджері після їх купівлі в Магазині.

```javascript
const playerState = {
  evo: 0,
  modules: {
    // 0D-1D: Вигляд згори (Текст + Базова аркада)
    top_down_view: true,       // Завжди доступно
    text_editor: true,         // Безкоштовно
    tile_grid: false,          // 🔒 300 EVO
    pixel_editor_basic: false, // 🔒 150 EVO
    
    // 2D: Вигляд збоку + Гравітація
    side_scroll_view: false,   // 🔒 400 EVO
    gravity_physics: false,    // 🔒 500 EVO
    jump_mechanics: false,     // 🔒 200 EVO
    
    // 3D: Всі ракурси + Вокселі
    axis_z_depth: false,       // 🔒 1000 EVO
    orbital_camera: false,     // 🔒 300 EVO
    fps_camera: false,         // 🔒 800 EVO
    css_3d_renderer: false,    // 🔒 500 EVO
  },
  highestStage: 1,
};
```

## 4. Етапи Еволюції та Модулі

### ЕТАП 1: 0D-1D (Вигляд згори / Текст / Логіка)
* **Сцена:** Текстовий фон або статична картинка. 
* **Логіка:** Програматор змінних (`hp`, `gold`). Умовні переходи між сценами (Кнопки).
* **Валідатор дерева квесту:** Перевірка "глухих кутів" та логічних розривів (безкоштовно).
* **Обмеження об'єктів:** До 10 сцен, до 5 параметрів на старті.

### ЕТАП 2: Аркада (Вигляд згори / Рух)
* **Сцена:** Сітка тайлів (Tile Grid) від 8x8 до 32x32.
* **Редактор:** Піксель-арт редактор спрайтів.
* **Контролер:** Рух по клітинках (WASD / D-pad / Swipe).
* **Матричний процесор:** Таймери та спавн для ігор типу Match-3 або Змійка.

### ЕТАП 3: Платформер (Вигляд збоку / 2D)
* **Фізика:** Поява модуля гравітації та вектора падіння.
* **Направлена колізія:** 
  * Верх: знищення ворога.
  * Низ: стояння на платформі.
  * Боки: зіткнення зі стіною.
* **Керування:** D-Pad + Кнопка стрибка (Jump Force, Coyote Time).

### ЕТАП 4: Воксельний Всесвіт (3D / Всі ракурси)
* **Простір:** 2D сітка еволюціонує у 3D масив `[x][y][z]`.
* **Рендер:** CSS 3D (`transform-style: preserve-3d`). Кожен воксель — це 6 граней DOM-елементів.
* **Камери:** Розблокування орбітальної камери (Drag to rotate) та FPS камери (Virtual Joystick).

## 5. Економіка та Ігрова Петля (Game Loop)

### Валюта: EVO (Бали Еволюції)
Єдина універсальна валюта платформи.

### Цикл взаємодії:
1. **Гра** → Гравець проходить чужі ігри та отримує EVO за активність.
2. **Пасивний дохід** → Створені гравцем ігри генерують EVO, коли в них грають інші.
3. **Еволюція** → Витрата EVO на розблокування модулів рушія (2D, 3D, камери, кольори).
4. **Скіни (Економіка творців)** → Створення кастомних текстур для чужих ігор. Дохід ділиться: 70% автору скіна / 30% автору гри.

## 6. Технічний Стек та Оптимізація під малу кількість об'єктів

* **Frontend:** HTML5, CSS3, Vanilla JS (ES6+). Відсутність важких фреймворків для миттєвого завантаження.
* **Зберігання:** `localStorage` (локально) + LZ-URL стиснення для шерінгу + GitHub Raw для бази ігор.
* **Мобільна адаптація:** Обов'язковий сенсорний HUD (D-pad, Joystick), жести (свайпи), AudioContext resume по першому тапу.

### Обмеження та продуктивність (CSS 3D Constraints)
Оскільки рушій використовує DOM для 3D рендеру (CSS 3D Transforms) та свідомо розрахований на малу кількість об'єктів для високої продуктивності, впроваджуються жорсткі архітектурні ліміти:
* **Ліміти сцени:** Максимум 16x16x16 вокселів (Десктоп) / 8x8x8 (Мобільні пристрої).
* **DOM-оптимізація:**
  * **Face Culling:** Грані вокселів, що торкаються одна одної або закриті сусідніми блоками, не генеруються та видаляються з DOM.
  * Застосування `backface-visibility: hidden` та `will-change: transform`.
  * `contain: layout style paint` для ізоляції перемальовок.
* **Динамічна якість:** При падінні FPS < 20, рушій автоматично зменшує `perspective` або вимикає тіні.

## 8. План Реалізації (Roadmap)

* **Фаза 0: PoC CSS 3D.** Тест лімітів DOM на цільових мобільних пристроях. Створення оптимізованого генератора граней (Face Culling).
* **Фаза 1: Текстовий движок (Вигляд згори).** Редактор логіки, сцени, валідатор дерева квестів. Створення еволюційного State-менеджера.
* **Фаза 2: 2D Аркада.** Тайлова сітка, контролери руху, піксельний редактор.
* **Фаза 3: Платформер (Вигляд збоку).** Вектор гравітації, направлені колізії, налаштування камери side-scroll.
* **Фаза 4: 3D Вокселі (Всі ракурси).** Розробка ядра редактора, інтеграція пошарового режиму, впровадження осі Z, орбітальної та FPS камер. Текстурування CSS граней та розміщення сутностей.

## 9. Детальна специфікація: Воксельний 3D-редактор (Етап 4)

### 9.1 Інтерфейс та інструменти редактора (Mobile-First)
Екран редактора поділено на три ключові зони:
1. **Панель інструментів (зверху):** Олівець (додати), Ластик (видалити), Відро (заливка), Виділення, Скасувати/Повторити (Undo/Redo до 30 кроків), Режим камери.
2. **Вікно 3D-перегляду (центр, 80% екрану):** Робоча область.
3. **Нижня панель вкладок:**
   * **Шари:** Пошаровий режим редагування (відображення поточного Z-рівня, приховування сусідніх шарів).
   * **Палітра:** Вибір блоку. Базово 16 кольорів. Кожен блок має властивості (`solid`, `transparent`, `light_source`, `damage`, `trigger`).
   * **Сутності:** Вставка об'єктів з попередніх етапів (гравець, вороги, тригери, збиральні предмети).
   * **Тест:** Швидкий запуск симуляції без виходу з редактора (з урахуванням гравітації та колізій).

### 9.2 Модель даних сцени (VoxelScene)
Дані сцени зберігаються у форматі JSON зі стисненням `lz-string` (до ~5 КБ):
```javascript
const VoxelScene = {
  meta: {
    id: "string", name: "Мій 3D-світ", author: "playerId", version: 1,
    viewMode: "3d", // "topdown" | "side" | "3d"
    orientation: "xz", // площина підлоги для side-view
    initialCamera: { x, y, z, targetX, targetY, targetZ }
  },
  grid: {
    sizeX: 16, sizeY: 16, sizeZ: 16, // До 8-16 для моб., до 32 для десктопу
    voxels: [] // { type: "solid", color: "#aabbcc", properties: {...} }
  },
  entities: [ // Сутності з Етапів 1-3
    { id: "player_start", type: "player", position: { x: 1, y: 1, z: 0 }, variables: { hp: 100 } }
  ],
  triggers: [
    { id: "dialog_trigger", position: {x:3,y:0,z:2}, radius: 1, action: "showDialog" }
  ],
  logic: { dialogs: { ... }, globalVariables: { key_found: false } }
};
```

### 9.3 Критерії приймання 3D-редактора
1. **Створення світу:** Користувач може створити 3D-світ розміром до 16×16×16 (десктоп) або 8×8×8 (мобільний).
2. **Перегляд:** Підтримка трьох режимів (Top-Down, Side-Scroller, 3D Perspective з FPS та орбітальною камерами).
3. **Сутності:** Можливість додавати ігрові об'єкти (гравець, вороги, тригери) та задавати їм властивості.
4. **Тестування:** Запуск симуляції рівня (рух, стрибки, тригери) безпосередньо в редакторі.
5. **Продуктивність:** Забезпечення 20+ FPS на мобільних, 30+ на десктопі завдяки Face Culling та `contain: layout`.
6. **Керування:** Повна підтримка сенсорного управління та історії дій (Undo/Redo).
